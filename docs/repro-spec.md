# 汎用歌枠DB 再現仕様書

## 1. 文書の目的

本書は、Googleスプレッドシートで管理されている歌枠データを、R2上の静的JSONとして配信し、HTMLで検索・閲覧できる索引型データベースとして再現するための仕様書です。

この文書は、以下の用途を想定します。

- 人間が構成を理解する
- 実装担当者が要件を誤解しない
- CODEX / GEMINI 等のAIに読み込ませて再現実装させる
- 将来の別案件へ流用する

---

## 2. このアプリの本質

このアプリは、**静的JSON優先**の歌枠索引データベースです。

基本構造は以下の通りです。

1. Googleスプレッドシートで元データを管理する
2. 必要なデータをJSONとして整形する
3. JSONをR2に配置する
4. 静的HTMLがJSONを読み込んで描画する
5. エンドユーザーはブラウザ上で検索・閲覧する

本アプリは、動的Webアプリやバックエンド主導の検索システムではありません。  
あくまで、**スプレッドシート管理情報を視認性高く検索できる静的索引DB** として設計します。

- JSON生成用スクリプト
- R2バケット

### 4.2 データフロー

## 6. スプレッドシート要件

### 6.1 基本方針
スプレッドシートは本システムの一次情報源です。  
そのため、列定義・値の形式・空欄ルールを曖昧にしないことが重要です。

### 6.2 想定シート
例:
- `songs`
- `archives`
- `meta`
- `gags`  
※ 実案件に応じて増減可

### 6.3 songs シートの想定列
以下は例です。実案件に応じて確定してください。

| 列名 | 内容 | 必須 |
|---|---|---|
| id | 一意識別子 | 推奨 |
| title | 曲名 | 必須 |
| artist | アーティスト名 | 必須 |
| artist_reading | 読み仮名等 | 任意 |
| date | 配信日 | 推奨 |
| archive_id | 配信識別子 | 推奨 |
| archive_url | 配信URL | 任意 |
| notes | 備考 | 任意 |
| status | 公開状態等 | 任意 |

### 6.4 値のルール
以下を事前に定義してください。

- 空欄は `""` とするか `null` とするか
- 日付形式は `YYYY-MM-DD` に統一するか
- 全角半角をどう扱うか
- 記号・括弧・波ダッシュ等の正規化をどうするか
- 同一曲の表記ゆれをどう扱うか

### 6.5 禁止事項
- 列名の無断変更
- 同じ意味の列を複数作ること
- 途中から日付形式を混在させること
- AIや人手での場当たり的なキー名変更

## 7. JSON出力仕様

### 7.1 基本方針
JSONは、HTMLがそのまま扱いやすい形で出力します。  
フロント側で複雑な再整形を前提にしないことを原則とします。

### 7.2 想定ファイル
- `public-data/songs.json`
- `public-data/archives.json`
- `public-data/meta.json`

### 7.3 songs.json の想定構造

```json
[
  {
    "id": "song-0001",
    "title": "曲名",
    "artist": "アーティスト名",
    "artistReading": "",
    "date": "2026-04-01",
    "archiveId": "archive-001",
    "archiveUrl": "https://example.com",
    "notes": "",
    "status": "published"
  }
]
```

### 7.4 設計原則
- JSONキー名は固定する
- キーの命名規則は統一する
- HTML側で期待するスキーマを明文化する
- 後方互換性を壊す変更はログと仕様書に残す

### 7.5 決めるべき点
- camelCase にするか snake_case にするか
- 空配列と空文字の使い分け
- 非公開データを含めるか
- ソート済みで出力するか

## 15. 実装ルール

### 15.1 原則
- 小さく作る
- 既存構造を崩さない
- スプレッドシート → JSON → R2 → HTML の流れを守る
- 変更は最小単位で行う
- 重要変更はログに残す

### 15.2 禁止
- 目的外の大規模リファクタ
- 理由なきライブラリ追加
- スキーマ変更の無断実施
- ドキュメント未更新のままの構造変更
